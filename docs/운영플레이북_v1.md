# 비즈필터 운영 플레이북 v1 (2026-06-11)

> 인입부터 납품·사후관리까지 전 구간 SOP. 1인 운영 기준. 모든 고객 발송 문구는 복붙 가능.
> 단일 기준 원칙: 48시간 제작 타이머는 **브리프 확정 시점**부터. 판정 합격선은 **결제클릭률 단일 지표**(CTR·CAC는 참고). 재검증 할인 30%.

## 핵심 결정 요약 (자주 묻는 것)
| 질문 | 결정 |
|---|---|
| 사업 이름(네이밍)은 누가? | 우리가 검증용 가칭 2~3개 제안, 고객은 선택만. 정식 브랜딩은 불포함 (B-4) |
| 사업 컨설팅 포함? | 불포함. 검증 설계 + 판정 해석 + 다음 액션 권고까지가 경계 (B-7) |
| 거창하고 막연한 아이디어는? | 킥오프 30분 + 브리프 1장으로 "첫 결제 한 건"으로 절단. 고객이 못 채우면 우리가 기본값 채우고 고객은 거부권만 (B-2, B-3) |

| demand gen vs 키워드 광고? | "이미 검색하는 문제인가" 10분 체크로 분기. 업종별 기본값 표 (C) |

| 고객 라이브 대시보드? | 초기에는 GA4 뷰어 초대 + 구글애즈 공유 보고서로 제공. 커스텀 대시보드는 수요 생기면 (D-3) |

## 섹션 A. 인입부터 결제까지 전환 SOP

운영 원칙: 모든 리드는 텔레그램 알림 수신 즉시 이 SOP대로만 처리합니다. 판단이 필요한 건 4번(자격 판정) 한 곳뿐이고, 나머지는 전부 복붙입니다.

---

### A-1. 응답 SLA

| 인입 시점 | 첫 응답 목표 | 규칙 |
|---|---|---|
| 평일 10~19시 | 30분 이내 (최대 2시간) | 텔레그램 알림 확인 즉시 채널별 첫 응답 템플릿 발송 |
| 평일 19시 이후 | 다음 날 10시 30분까지 | 당일 처리하지 않음. 밤에 답하면 기대치가 망가짐 |
| 주말/공휴일 | 다음 영업일 10시 30분까지 | 동일 |

시간 외 인입 대비 설정 (1회만 해두면 끝):
- [ ] 카톡 채널(@비즈필터) 자동응답 등록: "안녕하세요, 비즈필터입니다. 신청이 정상 접수되었습니다. 영업시간(평일 10~19시) 내 접수 건은 당일 중 순서대로 직접 답변드립니다. 영업시간 외 문의는 다음 영업일 오전에 연락드리겠습니다."
- [ ] 매일 아침 10시 루틴: 관리자 페이지 리드 목록 확인 → 미응답 건 응답 → 팔로업 시퀀스(A-7) 해당 건 발송. 하루 10분.

### A-2. 채널별 첫 응답 템플릿

개인화 규칙: 폼 데이터 3개를 반드시 본문에 넣습니다. ① 업종/아이디어 한 줄 ② 현재 단계 ③ 최우선 관심(퀴즈 답변). 이 3개가 들어가면 "자동발송 아니구나"가 전달되고, 그게 전환의 절반입니다.

#### 카톡 (기본 채널)

```
{이름}님, 안녕하세요. 비즈필터입니다.

남겨주신 내용 확인했습니다. {업종} 분야에서 "{아이디어 한 줄}"을
검토 중이시고, 현재 {단계} 단계에 계신 것으로 이해했습니다.
특히 {최우선 관심} 부분을 가장 궁금해하시는 것 같은데, 맞을까요?

정확한 안내를 위해 15분 정도 짧게 통화 또는 채팅 상담을
권해드립니다. 아래 중 편하신 시간을 알려주세요.

1. 오늘 {시간A}
2. 내일 {시간B}
3. 그 외 편하신 시간

상담은 무료이고, 상담했다고 진행하셔야 하는 것은 아닙니다.
```

#### 이메일 (카톡 ID 없이 이메일만 남긴 경우)

```
제목: [비즈필터] {이름}님 아이디어 검증 신청 확인드립니다

{이름}님, 안녕하세요. 비즈필터 대표 OOO입니다.

신청해주신 내용 잘 받았습니다. {업종} 분야의 "{아이디어 한 줄}",
현재 {단계} 단계로 확인했습니다.

진행 전 15분 상담으로 검증 설계가 가능한 아이디어인지 먼저
확인해드리고 있습니다. 아래 중 편하신 방법으로 회신 부탁드립니다.

1. 카카오톡 채널 추가 후 채팅: @비즈필터 (가장 빠릅니다)
2. 이 메일에 통화 가능 시간 회신

상담은 무료이며, 진행 여부는 상담 후 결정하시면 됩니다.
감사합니다.
```

#### 전화 (전화번호를 남긴 경우)

- 원칙: 바로 전화하지 않고 카톡/문자를 먼저 보낸 뒤 응답이 없으면 전화합니다. 콜드콜 느낌을 줄입니다.
- 선발송 문자: "{이름}님, 비즈필터입니다. 신청 확인 문자드립니다. 15분 상담 일정을 잡으려 하는데, 지금 통화 괜찮으시면 회신 주시고 편하신 시간이 따로 있으면 알려주세요."
- 통화 연결 시 오프닝: "안녕하세요 {이름}님, 비즈필터입니다. 신청서에 {업종} 아이디어로 {최우선 관심}이 궁금하다고 남겨주셔서 연락드렸습니다. 지금 15분 정도 괜찮으실까요?" → 괜찮다고 하면 바로 A-3 진행.

### A-3. 15분 상담 구조

순서대로 묻고, 답을 폼 데이터 옆에 한 줄씩 메모합니다. 상담의 목적은 설득이 아니라 자격 판정(A-4)과 플랜 결정(A-5)에 필요한 정보 수집입니다.

| # | 질문 | 목적 |
|---|---|---|
| 1 | "신청서에 적어주신 아이디어를 한 문장으로 다시 말씀해주시겠어요?" | 폼 내용과 실제 생각이 같은지 확인. 다르면 여기서 정리 |
| 2 | "이걸 돈 내고 쓸 사람은 구체적으로 누구라고 보세요?" | 타깃이 광고 타겟팅 가능한 수준인지 확인 |
| 3 | "그 사람들이 지금 겪는 문제가 뭔가요?" | 문제가 실재하는지, 광고 문구의 재료 확보 |
| 4 | "그 사람들은 지금 그 문제를 어떻게 해결하고 있나요?" | 현재 대안 파악. 대안이 없다는 답은 위험 신호 |
| 5 | "얼마를 받으실 생각인가요? 그 근거는요?" | 가격 가설 확보. 검증 사이트에 표시할 가격 결정 |
| 6 | "언제까지 결론이 필요하고, 검증에 쓸 수 있는 예산은 어느 정도인가요?" | 일정 긴급도와 Quick/Deep 적합성 판단 |
| 7 | "어떤 숫자가 나오면 진행, 어떤 숫자면 중단이라고 생각하세요?" | 합격선 사전 합의의 시작. 참고 기준(방문 100명 중 결제클릭 3명) 제시 |

7번 마무리 멘트 (그대로 사용):
"저희는 데이터를 보기 전에 합격선을 먼저 같이 정합니다. 참고 기준은 방문자 100명 중 결제 클릭 3명입니다. 결과가 나온 뒤에 기준을 움직이면 검증이 의미가 없어지기 때문에, 착수 전에 이 숫자를 확정하고 시작합니다."

### A-4. 자격 판정 기준

상담 종료 전에 셋 중 하나로 분류합니다. 보류하지 않습니다.

#### 즉시 진행 가능
- 온라인으로 결제 의향 측정이 가능한 구조
- 타깃이 광고로 도달 가능
- 합격선 합의에 동의
- 7일(또는 14일) 일정과 비용 수용
→ 바로 A-5 플랜 제안으로 넘어갑니다.

#### 설계 조정 필요 (오프라인/지역 사업 등)
- 기준: 매장, 지역 서비스, 대면 사업 등 전국 단위 온라인 검증이 안 맞는 경우
- 안내 멘트: "가능합니다. 다만 {이름}님 사업은 전국 광고가 아니라 {지역} 타겟 광고로 돌리고, 결제 버튼 대신 사전예약 신청을 측정 지표로 설계합니다. 측정 방식만 다르고 판정 구조는 동일합니다."

#### 정중히 거절
| 사유 | 기준 | 거절 멘트 (그대로 사용) |
|---|---|---|
| 법령 위반 소지 | 무허가 의료/금융, 사행성, 표시광고법 위반이 필요한 구조 | "솔직히 말씀드리면 이 아이템은 광고 심사와 관련 법령상 저희가 검증을 설계해드릴 수 없는 영역입니다. 비용을 받고 진행해도 광고가 집행되지 않아 판정 자체가 불가능합니다. 이번에는 도와드리기 어렵습니다." |
| 검증 불가 구조 | 입찰/조달, 6개월 이상 영업 사이클 등 7~14일 광고로 신호가 안 잡히는 모델 (단, 7~14일 내 유효 리드 측정이 가능한 B2B는 거절이 아니라 섹션 C의 B2B 기준으로 진행) | "이 사업은 고객 결정이 광고 7일로 잡히는 구조가 아니라서, 저희 방식으로는 의미 있는 판정을 드릴 수 없습니다. 판정을 못 드릴 일을 돈 받고 시작하지 않는 것이 저희 원칙입니다." |
| 비현실 기대 | 검증이 아니라 성공 보장이나 매출을 기대, 또는 No-Go 결과를 수용 못 함 | "저희가 드리는 것은 진행/중단/방향수정 판정이지, 성공 보장이 아닙니다. 중단이라는 답이 나올 수 있고 그것도 정상 납품입니다. 이 전제가 맞지 않으면 진행하셔도 서로 만족스럽지 않을 것 같습니다." |

거절 후 공통 클로징: "아이디어가 더 구체화되거나 구조가 바뀌면 그때 다시 연락 주세요. 기록은 남겨두겠습니다."

### A-5. 플랜 제안 스크립트

#### 기본 원칙: Quick 우선
첫 고객 접점에서는 항상 Quick(50만원/7일)을 먼저 제안합니다. Deep을 먼저 꺼내면 비교 대상이 없어 비싸게만 들립니다.

기본 제안 멘트:
"{이름}님 케이스는 Quick으로 충분합니다. 7일에 50만원이고, 광고비 5만원이 포함되어 있습니다. 검증용 사이트 제작, 구글 광고 7일 집행, 결제 의향과 CAC 측정, 그리고 판정 리포트와 30분 미팅까지입니다. 착수금 입금 후 48시간 안에 사이트와 광고를 띄우고, 나머지 닷새는 시장이 답하는 시간입니다."

#### Deep(130만원/14일)을 제안하는 신호 (하나라도 해당하면 언급만, 강권 금지)
- 가격을 두 가지 안 사이에서 고민 중 (가격 2안 테스트 필요)
- 경쟁사가 많아 포지셔닝 자체가 불확실
- 타깃이 인스타/페북 성향 (구글 검색 수요가 약한 B2C 감성 상품)
- 단가/마진 구조를 모름 (손익 시뮬 필요)
- 예산 여유가 있고 "한 번에 깊게" 성향

제안 멘트: "말씀 들어보니 가격을 {A안}과 {B안} 중에 못 정하고 계신데, 이 경우 Deep에서 두 가격을 동시에 테스트하면 어느 쪽이 팔리는 가격인지 데이터로 나옵니다. 다만 Quick으로 먼저 수요 자체를 확인하고 필요하면 올라가셔도 됩니다."

#### 가격 이의 응대 3종

1. "비싸다"
"50만원이 아깝게 느껴질 수 있습니다. 비교 대상은 검증 없이 만들었을 때의 비용입니다. 개발 외주 최소 수백만원과 몇 달의 시간을 쓰기 전에, 50만원과 7일로 그 돈을 써도 되는지 먼저 확인하는 구조입니다. 그리고 광고 미집행 등으로 분명한 판정을 못 드리면 전액 환불입니다(실제 집행된 광고비는 제외)."

2. "직접 해보겠다"
"직접 하실 수 있습니다, 맞습니다. 다만 사이트 제작, 광고 세팅, 측정 설계까지 직접 하시면 보통 3~4주가 걸리고, 측정 설계가 어긋나면 잘못된 결론이 나옵니다. 저희 값어치는 도구가 아니라 7일 안에 믿을 수 있는 판정이 나오는 설계입니다. 그래도 직접 해보시겠다면 말리지 않겠습니다. 막히는 지점이 생기면 그때 오셔도 됩니다."

3. "생각해보겠다"
"네, 충분히 고민하고 결정하세요. 한 가지만 여쭙겠습니다. 지금 결정을 미루는 이유가 비용인가요, 아니면 검증이 필요한지 자체가 확신이 안 서서인가요?" (답에 따라 1번 또는 기본 제안으로 복귀) "이번 주 {요일}까지 착수 슬롯이 하나 비어 있어서, 그 안에 결정 주시면 {날짜}에 바로 시작 가능합니다."
(주의: 슬롯 멘트는 사실일 때만 사용. 거짓 긴급은 금지)

### A-6. 결제 안내

#### 계좌이체 안내 멘트
```
{이름}님, 진행 결정 감사합니다. 결제 안내드립니다.

상품: 비즈필터 {Quick/Deep} ({금액}원, 광고비 {5만/20만}원 포함)
입금 계좌: {은행} {계좌번호} (예금주: {예금주명})
입금자명: {이름}님 성함으로 부탁드립니다.

입금 확인 후 24시간 안에 킥오프 통화를 진행하고,
검증 설계(브리프)가 확정된 시점부터 48시간 안에
사이트와 광고를 라이브합니다.

참고로 환불 기준을 미리 말씀드립니다.
1. 착수 전 취소: 전액 환불
2. 광고 미집행 등으로 분명한 판정을 못 드린 경우: 전액 환불
```

### B-1. 킥오프 타이밍과 형식

**규칙**

- [ ] 입금 확인 즉시(영업시간 내 1시간 안에) 킥오프 일정 제안 메시지 발송

```
[비즈필터] 입금 확인되었습니다. 감사합니다.

예) 오늘 오후 3시 / 내일 오전 10시

통화에서 정하는 것은 딱 4가지입니다.
1. 누구에게 무엇을 파는지 한 문장
2. 테스트 가격
3. 합격선 숫자
4. 검증용 가칭(저희가 2~3개 제안드립니다)

통화 전에 참고할 자료(유사 서비스 링크, 보유 사진 등)가
```

**30분 아젠다 (시간 엄수)**

### B-2. 검증 브리프 1장 양식

브리프는 한 장을 넘기지 않는다. 모든 필드는 "고객이 못 채우면 우리가 기본값으로 채우고, 고객은 거부권만 행사"하는 구조다.

| 필드 | 정의 | 고객이 못 채울 때 기본값 규칙 |
|---|---|---|
| 타깃 한 줄 | "누가"를 한 줄로. 나이/직업/상황 중 2개 이상 포함 (예: 이직 준비 중인 5년차 직장인) | 신청폼 아이디어 설명에서 가장 구체적인 사용자상을 추출. 추출 불가 시 "이 문제를 돈 내고 해결할 확률이 가장 높은 집단"으로 우리가 지정 |
| 문제 | 타깃이 지금 돈/시간을 쓰며 겪는 불편 한 줄. "있으면 좋은 것"은 문제가 아님 | 고객이 말한 것 중 빈도가 가장 높고 지불 이력이 있는 불편 하나만 채택. 나머지는 "제외할 것"으로 이동 |
| 현재 대안 | 그 문제를 지금 어떻게 해결하고 있는가 (경쟁사, 엑셀, 알바, 그냥 참음 등) | 네이버/구글 검색 10분으로 우리가 조사해 기입. 대안이 정말 없으면 "참거나 포기"로 기입 (대안 없음 = 시장 없음 신호일 수 있음을 브리프에 메모) |
| 오퍼 한 문장 | 사이트 헤드라인이 될 약속 한 문장. 기능 나열 금지, 결과 중심 | B-3 공식으로 통화 중 수렴. 안 되면 우리가 초안 2개 작성 후 택1 요청 |
| 가격 가설 | 사이트에 표시할 실제 가격 숫자 1개 (Deep은 2개) | B-5 규칙으로 우리가 산출해 제안 |
| 핵심 소구점 2~3개 | 광고 문구와 사이트 섹션이 될 구매 이유. 최대 3개, 초과분은 버림 | 현재 대안의 약점에서 역산해 우리가 작성 (더 싸다/더 빠르다/더 간편하다 중 해당되는 것) |
| 제외할 것 | 이번 검증에서 다루지 않는 기능/타깃/메시지 명시 | 고객이 말한 아이디어 중 "한 문장"에 안 들어간 전부를 여기로 옮긴다. 빈칸 금지 |
| 합격선 숫자 | 판정 기준. 기본: 방문자 100명당 결제 클릭 3명 | B-6 조정 원칙 적용. 고객 동의 문구를 카톡으로 받아 기록 |
| 광고 채널 | Quick: 구글 검색광고 기본. 오프라인/지역 사업: 지역타겟 + 사전예약 측정 | 검색량이 있으면 검색광고, 검색어 자체가 없는 신개념이면 디스플레이/지역타겟으로 우리가 결정하고 사유 한 줄 기입 |

운영 원칙: 기본값으로 채운 필드에는 브리프에 `(비즈필터 제안)` 표시를 남긴다. 나중에 판정 미팅에서 "그건 내 생각이 아니었다"는 분쟁을 막는 장치다.

### B-3. 아이디어 좁히기 공식

**수렴 공식 (통화 중 소리 내어 같이 채운다)**

> "[타깃]이 [문제]를 겪을 때, [현재 대안] 대신 [오퍼]를 [가격]에 쓰게 한다"

**진행법 (5~15분 구간)**

1. 고객에게 아이디어를 2분간 자유롭게 말하게 한다. 끊지 않는다
2. "방금 말씀에서 고객이 여러 종류 나왔는데, 딱 한 부류만 남긴다면 누구입니까?"로 타깃을 자른다
3. "그분들이 지금은 그 문제를 어떻게 해결하고 있나요?"로 현재 대안을 확보한다
4. 공식 문장을 직접 읽어준다. "정리하면, OO한 분이 OO할 때, 지금은 OO로 버티는데, 그 대신 OO를 O만원에 쓰게 하자는 겁니다. 맞습니까?"
5. 고객이 "맞는데 그것 말고도..."라고 하면: "그 부분은 브리프의 제외할 것 칸에 적어두겠습니다. 이번 7일은 이 한 문장만 시험합니다. 이게 통하면 다음 것도 시험할 수 있습니다"

**너무 큰 아이디어(플랫폼/슈퍼앱/마켓플레이스) 절단 규칙**

- 규칙: 양면 시장이면 돈 내는 쪽 하나만 남긴다. 기능이 3개 이상이면 가장 지불 의향이 높은 기능 1개만 남긴다
- 변환 질문: "이 사업에서 일어날 첫 번째 결제 한 건은, 누가 무엇에 얼마를 내는 장면입니까?" 그 장면 하나가 이번 검증 대상이다
- 고객 설득 멘트 (그대로 사용):

```
플랫폼은 첫 거래가 일어나야 플랫폼이 됩니다.
이번 7일은 전체 그림이 아니라, 첫 결제 한 건이
실제로 일어나는지만 확인합니다.
첫 거래가 증명되면 나머지 그림은 그 위에 쌓으시면 됩니다.
```

### B-4. 검증용 가칭과 도메인 규칙

**규칙**

- [ ] 가칭은 우리가 2~3개 제안하고 고객은 고르기만 한다. 고객에게 작명 숙제를 주지 않는다 (작명 회의로 하루를 잃는 것이 최대 리스크)
- [ ] 통화 전에 후보를 만들어 간다. 통화에서 즉석 작명 금지
- [ ] 도메인은 .kr 또는 .co.kr 또는 .com 중 즉시 구매 가능한 것. 가칭 그대로 안 되면 뒤에 일반어를 붙인다 (예: OOO.kr 불가 시 getOOO.kr, OOOservice.kr)
- [ ] 고객이 3개 다 싫다고 하면: 1회에 한해 추가 2개 제안. 그래도 결정 못 하면 우리가 1번 후보로 확정하고 진행 (브리프에 기록)

**작명 기준 3가지**

| 기준 | 설명 | 예시 방향 |
|---|---|---|
| 직관성 | 듣자마자 뭐 하는 서비스인지 추측 가능 | "타깃/문제 + 동사/일반명사" 조합 |
| 검색 가능성 | 한글 표기가 하나로 고정되고, 검색 시 대형 브랜드에 묻히지 않음 | 기존 유명 서비스명과 두 글자 이상 겹치지 않게 |
| 분쟁 회피 | 일반명사 2개 조합 위주, 유명 상표와 유사 발음 회피. 킵리스 상표검색(특허정보넷)에서 동일 명칭 등록 여부 1분 확인 | 동일 업종 등록상표와 겹치면 후보 탈락 |

**스코프 안내 멘트 (가칭 제시 직후 그대로 사용)**

```
이 이름은 검증을 위한 가칭입니다.
7일 동안 광고와 사이트에서 작동하는지만 보면 되기 때문에,
정식 브랜드명이나 로고 작업은 이번 범위에 포함되지 않습니다.
검증이 Go로 나오면 그때 정식 네이밍과 브랜딩을
별도로 진행하시는 것을 권해드립니다.
```

### B-5. 가격 가설 정하는 법

통화 전 준비 단계에서 15분 안에 끝낸다. 그 이상 쓰지 않는다.

**1순위: 유사 서비스 가격대 조사**

1. 현재 대안(직접 경쟁사 + 대체재) 3곳의 공개 가격을 찾는다 (네이버 검색, 크몽/숨고, 앱스토어, 경쟁사 사이트)
2. 3곳 가격의 중간값을 기준으로, 우리 오퍼가 대안보다 나은 점이 명확하면 중간값 그대로, 신규 무명 브랜드 페널티를 고려하면 중간값의 80~90% 선으로 잡는다
3. 끝자리는 9,000원/49,000원 식으로 정리

**2순위 (유사 서비스가 없을 때): 객단가 역산**

1. 고객에게 묻는다: "이게 사업이 되려면 한 명당 얼마를 받아야 한다고 생각하십니까?"
2. 그 숫자가 타깃의 지불 체감(타깃 월소득 대비, 또는 현재 대안에 쓰는 돈 대비)과 맞는지 점검한다. 현재 대안 지출의 3배를 넘으면 하향 제안
3. Deep 상품의 가격 2안 테스트: 1안 = 역산 가격, 2안 = 그 1.5~2배. (싼 가격 2개를 테스트하지 않는다. 위로 받을 수 있는지를 본다)

**기록 규칙**: 브리프 가격 칸에 산출 근거를 한 줄로 남긴다. 예: `49,000원 (크몽 유사 서비스 3건 중간값 55,000원의 90%)`

**조정 원칙 (표에서 해당 줄 하나만 적용, 중복 적용하지 않음)**

| 상황 | 합격선 조정 | 이유 |
|---|---|---|
| 기본 (5만원 미만 소비재/디지털) | 3/100 유지 | 표준 기준 |
| 객단가 10만원 이상 | 2/100 | 고가격은 클릭 문턱이 높음 |
| 객단가 50만원 이상 또는 B2B | 1/100 + 상담신청 3건 병행 측정 | 즉시 결제 의향보다 리드 품질이 신호 |

**반드시 지킬 것**: 합격선은 광고 시작 전에 확정하고, 카톡으로 서면 기록을 남긴다. 데이터를 본 뒤에 기준을 바꾸는 것은 어느 쪽도 불가하다.

**합격선 합의 메시지 (브리프 발송 시 함께 전송, 답장 필수)**

```

· 기준: 사이트 방문자 100명당 결제 클릭 OO명 이상
· 근거: (예: 객단가 12만원으로 고가형 기준 적용)

이 기준 이상이면 Go, 근접하면 Pivot 검토,
크게 미달하면 No-Go로 판정해 드립니다.

광고 시작 후에는 기준을 변경할 수 없습니다.
"동의합니다"라고 답주시면 이 기준으로 진행하겠습니다.
```

### B-8. 고객 제공물 체크리스트

킥오프 일정 제안 메시지와 함께 안내한다. 원칙: 어떤 항목도 진행의 전제조건이 아니다. 고객 자료를 기다리느라 48시간 제작이 늦어지면 안 된다.

| 항목 | 받으면 좋은 것 | 못 받을 때 대체 규칙 |
|---|---|---|
| 참고 링크 | 마음에 드는 유사 서비스/경쟁사 사이트 2~3개 | 우리가 리서치한 경쟁사 사이트로 대체 |
| 보유 사진/이미지 | 제품 사진, 작업 사진, 매장 사진 | 스톡 이미지 + AI 생성 이미지로 대체. 단 실물이 중요한 업종(음식/공간/제품)은 사진 1장이라도 받도록 한 번 더 요청 |
| 기존 자료 | 기획서, 메모, 인스타 계정 등 | 신청폼 답변 + 킥오프 통화 내용만으로 진행 |
| 사업자/연락 정보 | 사이트 하단 표기용 (선택) | 미기재 또는 "사전등록 페이지" 형태로 처리 |
| 카톡/연락 채널 | 사전등록자 응대 채널 | 비즈필터가 수집만 하고 명단을 검증 종료 시 전달 |

**자료 요청 메시지 (킥오프 일정 제안에 덧붙여 발송)**

```
준비되시면 아래 자료를 보내주세요.
없으셔도 진행에는 문제가 없습니다.

1. 참고하고 싶은 유사 서비스 링크 2~3개
2. 보유하신 사진 (제품, 작업, 매장 등)
3. 기존에 정리해두신 자료 (기획 메모 등)

보내주신 자료는 검증용 사이트 제작에만 사용합니다.
```

**자료 마감 규칙**: 킥오프 통화 후 6시간까지 도착한 자료만 반영한다. 이후 도착분은 "제작 완료 후 1회 수정" 때 반영한다. 이 규칙은 킥오프 통화 말미에 구두로 고지한다.

## 섹션 C. 업종/상황별 광고 채널 결정 트리

### C-1. 핵심 분기 질문: "사람들이 이 문제를 이미 검색하는가?"

채널 결정의 90%는 이 질문 하나로 끝납니다. 신청서 접수 후 10분 안에 아래 체크를 돌립니다.

**검색 수요 판별 체크리스트 (10분 컷)**

- [ ] 네이버 검색창에 핵심 문제 키워드 2~3개 입력, 자동완성이 뜨는가
- [ ] 구글 검색창에 같은 키워드 입력, 자동완성/연관검색어가 뜨는가
- [ ] 네이버 검색광고 키워드도구(searchad.naver.com)에서 월간 검색수 확인, 합산 월 1,000회 이상인가
- [ ] 네이버 지식iN에 해당 문제로 질문 글이 존재하는가 (최근 1년 내 글이면 가산점)
- [ ] 구글 키워드 플래너에서 관련 키워드 월 검색량이 잡히는가

**판정 규칙**: 5개 중 2개 이상 해당하면 "검색 수요 있음"으로 분류합니다. 1개 이하면 "검색 수요 없음"입니다. 애매하면 없음으로 칩니다. 검색광고는 수요가 확실할 때만 돈값을 합니다.

### C-2. 결정 트리

```
Q1. 사람들이 이 문제를 이미 검색하는가? (C-1 체크리스트)
│
├─ 예 → Q2. B2B 고단가 상품인가? (객단가 100만원 이상, 기업 고객)
│        ├─ 예 → [구글 검색광고 + 상세 자격질문 리드폼]
│        └─ 아니오 → [구글 검색광고 (키워드)]
│
└─ 아니오 → Q3. 오프라인/지역 기반 사업인가?
         ├─ 예 → [메타 또는 구글 지역타겟 + 사전예약 측정]
         └─ 아니오 → Q4. 시각적으로 보여줄 수 있는 B2C인가?
                  ├─ 예 → [메타(인스타) 디맨드젠]
                  └─ 아니오 → [구글 검색광고, 인접 키워드로 우회]
```

**각 분기의 근거 한 줄**

| 분기 | 근거 |
|---|---|
| 검색 수요 있음 → 구글 검색광고 | 이미 문제를 인지하고 해결책을 찾는 사람에게 노출하는 것이 전환까지 가장 짧은 경로입니다 |
| 수요 없음 + 시각 소구 B2C → 메타 디맨드젠 | 검색하지 않는 수요는 피드에서 시각적으로 발견시키는 수밖에 없습니다 |
| 지역 오프라인 → 지역타겟 + 사전예약 | 상권 반경 밖 클릭은 전부 버리는 돈이므로 지역 반경 타겟이 필수입니다 |
| B2B 고단가 → 구글 검색 + 자격질문 폼 | B2B는 클릭 수가 아니라 "진짜 구매 권한이 있는 리드 1건"이 신호이므로 폼에서 걸러야 합니다 |

### C-3. 업종별 기본값 표

| 업종 | 기본 채널 | 측정 지표 | 합격선 가이드 (사전 합의 기준) | 주의점 |
|---|---|---|---|---|
| SaaS/웹서비스 | 구글 검색광고 | 가격페이지 결제클릭 | 방문 100명당 결제클릭 3명 | 무료체험 클릭은 결제 의향이 아님, 가격 표시 필수 |
| 모바일앱 | 메타 디맨드젠 | 사전등록(유료 플랜 선택 포함) | 방문 100명당 유료 의향 등록 3명 | "출시 알림"만 받으면 신호 약함, 가격 선택지를 넣을 것 |
| 이커머스/스마트스토어 | 메타 디맨드젠 | 구매버튼 클릭 | 방문 100명당 구매클릭 3명, 클릭당 비용이 마진 이내 | CAC가 객단가 마진을 넘으면 클릭률이 좋아도 No-Go |
| 교육/클래스 | 검색 수요 있으면 구글, 없으면 메타 | 수강신청 클릭 또는 사전예약 | 방문 100명당 신청클릭 3명 | 시즌성(방학, 연초) 영향 큼, 판정 리포트에 명시 |
| 전문서비스(B2B) | 구글 검색광고 + 자격질문 폼 | 자격 통과 리드 수 | 7일 내 유효 리드 2~3건 (클릭률 아닌 리드 절대수로 합의) | 모수가 작으므로 % 기준 대신 건수 기준으로 사전 합의 |
| 오프라인 매장 | 메타 지역타겟 (보조: 구글 지역) | 사전예약/오픈알림 + 연락처 제출 | 방문 100명당 예약 3명, 단 상권 반경 내 클릭만 집계 | 반경 3~5km 타겟 필수, 광역 노출 수치는 판정에서 제외 |

합격선은 참고 기준이며, 어떤 경우든 데이터를 보기 전에 고객과 숫자를 합의하고 시작합니다.

### C-4. Quick 예산 규칙 (광고비 5만원)

- [ ] **단일 채널 집중**: 5만원을 두 채널로 쪼개면 양쪽 다 판정 불가 수준의 데이터만 남습니다. 결정 트리에서 나온 채널 하나에만 전액 투입합니다.
- [ ] **집행 일정**: 착수 후 48시간 내 사이트+광고 셋업 완료, 이후 5일 집행. 일예산 1만원 × 5일 = 5만원.
- [ ] **문구 배분**: 문구 2~3종을 같은 캠페인 안에서 균등 로테이션으로 시작합니다(구글애즈 "최적화 안 함: 무제한 순환게재" 설정). 자동 최적화는 데이터가 적을 때 한 문구로 쏠려서 비교가 깨집니다.
- [ ] **3일차 점검**: 클릭 0인 문구는 끄고, 남은 예산을 반응 있는 문구에 몰아줍니다.
- [ ] **예외**: B2B는 CPC가 높아 5일간 클릭 30~50회 수준일 수 있습니다. 사전에 "클릭 수가 아니라 유효 리드 건수로 판정한다"고 고객과 합의합니다.

### C-5. 광고 문구 2~3종 설계 규칙

**소구점 분리 원칙**: 한 문구에는 소구점 하나만 넣습니다. 섞으면 어느 소구가 먹혔는지 알 수 없게 되어 검증 가치가 사라집니다.

| 문구 | 소구점 | 예시 골격 |
|---|---|---|
| A안 | 문제 소구 | "{고통스러운 상황}, 아직도 {기존 방식}으로 하세요?" |
| B안 | 가격 소구 | "{핵심 가치}를 월 {가격}원에" |
| C안 | 신뢰 소구 | "{권위/근거/숫자}가 만든 {카테고리}" |

**반응 없을 때 해석법 (판정 리포트에 그대로 사용)**

| 관찰 결과 | 해석 | 결론 |
|---|---|---|
| 노출은 충분한데 전 문구 클릭률 저조 (검색광고 CTR 2% 미만, 메타 1% 미만) | 시장이 이 문제 자체에 관심이 없음 | 아이디어 문제 쪽, No-Go/Pivot 신호 |
| 특정 문구만 클릭률 양호, 나머지 저조 | 그 소구점이 시장의 언어 | 문구 문제였음, 이긴 소구로 메시지 고정 |
| 클릭률은 양호한데 사이트에서 결제클릭 없음 | 관심은 있으나 오퍼(가격/구성)가 약함 | 아이디어/가격 문제, Pivot 신호 |
| 클릭률도 결제클릭도 합격선 이상 | 수요와 오퍼 모두 검증됨 | Go |

핵심 규칙: **문구 2~3종이 전부 죽으면 아이디어 문제, 하나라도 살면 문구 문제**입니다. 이래서 소구점을 분리해 2~3종을 돌립니다.

### C-6. 채널 결정 고객 설명 템플릿 (복붙용)

> 안녕하세요, 비즈필터입니다. {아이디어명}의 광고 채널을 {선택 채널}로 설계했습니다. 이유를 말씀드리면, {네이버/구글 검색 데이터 확인 결과 한 줄. 예: "관련 키워드의 월간 검색량이 약 N회로 확인되어, 이미 해결책을 찾고 있는 분들에게 직접 노출하는 검색광고가 가장 빠른 검증 경로입니다" 또는 "검색 수요가 거의 잡히지 않아, 인스타그램 피드에서 시각적으로 발견시키는 방식이 적합합니다"}. 이번 검증에서는 광고비 전액을 이 채널 하나에 집중해 {측정 지표}를 측정하며, 합격 기준은 사전에 합의드린 {합의 기준 숫자}입니다. 데이터가 쌓이는 대로 대시보드 링크로 공유드리겠습니다.

### D-4. 판정 미팅 30분 아젠다

사전 준비: 미팅 2시간 전까지 리포트 발송 완료.

| 시간 | 내용 |
|---|---|
| 0~5분 | 숫자 요약: 노출, 방문, CTR, 결제클릭, CAC. 해석 없이 사실만 |
| 5~10분 | 합격선 대비: Day1에 서면 합의한 기준 표를 띄우고 항목별 통과/미달 표시 |
| 10~18분 | 해석: 왜 이렇게 나왔나. 단계별 신호(광고 반응 → 사이트 체류 → 결제클릭) 중 어디서 끊겼는지 |
| 18~22분 | 판정 선언: Go / No-Go / Pivot 한 문장으로 명확하게. 얼버무리지 않음 |
| 22~30분 | 다음 액션 옵션 안내 + 질의응답 |

판정별 멘트 톤 가이드:

**Go (담백하게, 들뜨지 않게. 수익 보장처럼 들리는 표현 금지)**

```
합의한 기준을 통과했습니다. 만들 가치가 있다는 신호입니다.
다만 이것은 "수요가 있다"는 확인이지 성공 보장이 아닙니다. 다음 단계는 실제 제품으로 이 수요를 받아내는 일입니다.
저희는 개발을 하지 않습니다. 사이트와 데이터 일체를 이관해 드리고, 원하시면 개발 파트너를 소개만 해 드립니다. 소개에 따른 수수료는 받지 않습니다.
```

**Pivot (가능성은 인정하되 조건을 명확히. 재구매 권유가 영업처럼 들리지 않게 근거 먼저)**

```
전체 기준은 미달이지만, {특정 구간}의 신호는 살아 있습니다.
{가격/타겟/메시지}를 바꾸면 결과가 달라질 수 있는 구조입니다.
조건을 바꿔 재검증을 원하시면, 사이트와 데이터가 이미 있어 작업량이 적은 만큼 재검증은 30% 할인된 금액으로 진행해 드립니다. 다만 재검증 역시 결과를 보장하지는 않습니다. 결정은 천천히 하셔도 됩니다.
```

**No-Go (위로하되 단호하게. No-Go가 손해가 아니라 산출물임을 숫자로 증명)**

```
이 조건으로는 진행하지 않으시는 것이 맞다는 판정입니다.
아쉬운 결과지만, 이 답을 지금 아신 것이 이번 검증의 산출물입니다.
만약 검증 없이 만드셨다면 개발비와 시간으로 최소 {N}백만원, {N}개월이 들어갔을 일입니다.
50만원으로 그 손실을 막으신 것이고, 저희가 드린 판정은 명확합니다.
다음 아이디어가 생기면 그때 또 검증부터 하시면 됩니다.
```

### D-5. 판정 리포트 목차 (작성 타임박스: 90분, 절대 초과 금지)

분량은 4~6페이지. 노션 또는 PDF. 템플릿 1개를 만들어 두고 숫자와 해석만 교체합니다.

1. **최종 판정** (첫 페이지, 한 문장: Go / No-Go / Pivot) (5분)
2. **종합 신호 강도** (강함/보통/약함 3단계 + 한 줄 근거) (5분)
3. **핵심 지표**: CTR, 방문자, 결제클릭률, CAC 1차 추정치, 합의 합격선 대비 표 (20분)
4. **단계별 신호**: 광고 반응 → 방문 행동 → 결제 의향, 어디서 통과하고 어디서 끊겼는지 (20분)
5. **시장·경쟁 조사 요약** (Day1 미니 리서치 재정리, 신규 조사 금지) (10분)
6. **인사이트 3개 이내** (숫자에 근거한 것만) (20분)
7. **권고**: 판정별 다음 액션 1~3개 (10분)

90분이 넘으면 인사이트를 줄이지, 시간을 늘리지 않습니다. 리포트의 가치는 두께가 아니라 판정의 명확함입니다.

### D-6. 납품/이관 절차 (Go 판정 시 또는 고객 요청 시, 총 30~40분)

- [ ] 도메인: 고객 명의 계정으로 이전 신청 (또는 고객이 원치 않으면 만료 후 소멸 고지) (10분)
- [ ] 사이트: 코드 zip 또는 배포 계정 이전, 호스팅 안내 (10분)
- [ ] 데이터: GA4 속성 관리자 권한 이전, 광고 성과 CSV 내보내기, 사전등록자 명단(이메일/연락처) 전달 (10분)
- [ ] 광고 계정: 캠페인 중지 확인, 원하면 캠페인 구조 공유 (5분)
- [ ] 이관 완료 확인 메시지 발송

> 이관 완료 확인 메시지

```
{고객명}님, 이관이 완료되었습니다. 전달 내역은 아래와 같습니다.

1. 도메인 {도메인}: 고객님 계정으로 이전 완료
2. 사이트 원본 파일: {전달 방법}
3. GA4 데이터: 관리자 권한 이전 완료
4. 광고 성과 데이터: CSV 첨부
5. 사전등록자 명단 {N}건: 첨부

각 항목 접근이 되는지 확인 부탁드립니다. 확인 회신을 주시면 프로젝트를 종료 처리하겠습니다.
이후 개발 파트너 소개가 필요하시면 언제든 말씀해 주세요.
```

### D-7. 케이스 공개 동의 요청 (판정 미팅 직후 또는 리포트 발송 시)

```
{고객명}님, 한 가지 부탁드릴 것이 있습니다.
이번 검증 과정과 결과를 비즈필터 사례로 소개하고 싶습니다. 다음 두 가지 중 선택하실 수 있습니다.

1) 익명 공개: 업종과 숫자만 공개, 아이디어 내용과 신원은 비공개
2) 실명 공개: 프로젝트명과 함께 공개 (공개 전 원고를 먼저 확인시켜 드립니다)

물론 거절하셔도 전혀 무관하며, 서비스에는 어떤 영향도 없습니다.
편하게 답 주시면 됩니다.
```

### D-8. 사후 관리

#### 종료 7일 후 팔로업 (캘린더에 종료일+7일로 등록, 발송 3분)

> Go 고객용

```
{고객명}님, 비즈필터입니다. 검증 종료 후 일주일이 지나 안부 겸 연락드립니다.
다음 단계 진행은 잘 되고 계신지요. 이관 자료(계정 접근, 파일) 관련 문제는 이관 후 14일까지 무상으로 지원해 드립니다.
혹시 새로운 아이디어가 생기시면, 기존 고객님께는 재검증 30% 할인을 적용해 드립니다.
```

> No-Go / Pivot 고객용

```
{고객명}님, 비즈필터입니다. 일주일이 지나 연락드립니다.
지난 검증으로 큰 비용이 들어가기 전에 방향을 확인하셨으니, 다음 아이디어는 더 가볍게 시도하실 수 있습니다.
다음 검증은 30% 할인으로 진행해 드리며, Pivot 제안드렸던 조건 변경 재검증도 동일하게 적용됩니다.
```

> 후기 요청 (팔로업 회신이 긍정적일 때만 이어서 발송)

```
감사합니다. 한 가지만 더 부탁드리면, 이번 검증 경험을 2~3줄 후기로 남겨 주실 수 있을까요?
"무엇이 가장 도움이 되었는지" 한 가지만 적어 주셔도 충분합니다.
후기는 저희 사이트에 익명으로 게재되며, 원치 않으시면 게재하지 않습니다.
```

#### 환불 발생 시 처리 순서 (판정 보장형 기준, 당일 처리 원칙)

1. **사유 분류** (10분): 착수 전 청약철회인지, 판정 불가(광고 미집행/데이터 미확보)인지, 단순 불만(No-Go 결과 불만 등)인지 구분
2. **환불 대상 확정**: 착수 전 철회는 전액. 판정 불가는 광고비 실집행분 제외 전액. No-Go 불만은 환불 사유 아님 (판정은 납품된 결과물)
3. **고객 안내 발송** (아래 템플릿)
4. **계좌이체 환불** (영업일 3일 내, 입금 확인 후 완료 메시지)
5. **내부 기록**: 리드 상태 변경 + 원인 한 줄 회고 (같은 원인 2회 발생 시 SOP 수정)

> 환불 안내 템플릿 (판정 불가 케이스)

```
{고객명}님, 이번 검증에서 {사유: 광고 집행 불가 / 데이터 미확보}로 인해 분명한 판정을 드리지 못했습니다.
판정 보장 원칙에 따라 전액 환불 처리하겠습니다.
다만 이미 실집행된 광고비 {N}원은 환불 범위에서 제외되며, 환불 금액은 {N}원입니다.
영업일 3일 내 {계좌}로 입금해 드리고, 완료 후 다시 안내드리겠습니다. 불편을 드려 죄송합니다.
```

> No-Go 결과에 대한 환불 요청 응대 템플릿

```
{고객명}님, 말씀 주신 내용 확인했습니다.
비즈필터의 환불 기준은 "분명한 판정을 드리지 못한 경우"입니다.
이번 검증은 광고가 정상 집행되었고 7일치 데이터가 확보되어 No-Go라는 명확한 판정을 납품해 드렸습니다.
No-Go는 실패가 아니라, 큰 비용이 들어가기 전에 확인된 답입니다. 이 점 너른 양해 부탁드립니다.
추가로 데이터 해석에 궁금한 점이 있으시면 언제든 설명드리겠습니다.
```

## 섹션 E. 예외 상황 처리 규칙 (감사 반영)

| 상황 | 제안 규칙 |
|---|---|
| 입금 전 잠수 | 결제 안내 발송 시 입금 기한 3영업일 명시. 기한 1일 전 리마인드 1회, 기한 경과 시 A-7 시퀀스로 전환하고 슬롯 해제 |
| 광고 심사 반려 | Day2 라이브 체크에 반려 분기 추가. 1차: 문구 수정 후 재심사(당일). 2차 반려: 고객에게 일정 1~2일 순연 고지. 정책상 집행 자체가 불가로 판명: 판정 불가로 분류, D-8 환불 절차 즉시 가동 |
| 킥오프 후 고객 잠수 | 브리프 확답 요청 후 24시간 무응답 시 리마인드 1회. 48시간 무응답 시 기본값 브리프로 진행 고지(B-2의 거부권 구조 활용). 그래도 무응답이면 타이머 정지와 일정 순연을 서면 고지 |
| 표본 미달 | 광고비 5만원으로 방문 100명이 안 모이는 고CPC 업종 대응이 없음. 합격선 합의 시 최소 표본(예: 방문 70명)을 같이 합의하고, 미달 시 비율 환산 판정 또는 1~2일 연장 중 택1을 사전 명시. 이게 없으면 "표본이 적어 판정이 아니다"라는 불복과 환불 요구의 근거가 됩니다 |
| 도메인 비용 주체 | 도메인 1년 등록비는 상품가에 포함(상한 2만원, 초과분은 고객 부담 사전 안내). Go 이관 시 이전 수수료는 고객 부담, No-Go 시 만료 소멸은 D-6에 이미 있음 |
| 세금계산서/현금영수증 | 어디에도 없음. 가격의 부가세 포함 여부부터 확정 필요. 규칙: 사업자등록 완료 전에는 계약 보류 또는 등록 직후 소급 발행 약속을 서면으로. 등록 후에는 세금계산서/현금영수증 중 택1 발행 |

```
{이름}님, 어제 안내드린 결제 건 리마인드드립니다.
```
