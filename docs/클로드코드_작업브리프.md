# Claude Code 작업 브리프 — "아이디어 → 첫 결제" 런칭 파트너

> **이 문서 용도:** Claude Code에 넘겨 바로 작업 시작용.
> 레포 루트에 `CLAUDE.md`로 저장하거나, 첫 세션에 통째로 붙여넣기.
> (전략 상세는 별도 문서 `사업런칭파트너_시스템정리.md` 참고 — 이 브리프는 *빌드 시작*에 필요한 것만.)

---

## 1. 한 줄 요약

비개발자/예비창업자의 사업 아이디어를 **검증(실제 광고+데이터) → MVP 제작 → 첫 결제까지** 대신 해주는 DFY(Done-For-You) 런칭 서비스.

**지금 만들 것 (첫 빌드):** 이 *서비스 자체*의 랜딩 페이지 + 리드/의도 캡처 + 애널리틱스.
→ 서비스 수요를 먼저 테스트하는 용도 + 나중에 클라이언트 검증에 그대로 재사용할 템플릿.

---

## 2. 핵심 포지셔닝 (카피·UX 전부 여기서 나옴)

- **한 줄:** "의견 말고 **증거**를 드립니다. 실제 광고로 진짜 고객 반응을 눈앞에서 확인하고, 안 될 건 빨리 접고, 될 것만 만들어 첫 고객까지."
- **차별점 2개:**
  1. **검증 우선** — "안 될 것 같으면, 만들지 말라고 말해주는 유일한 곳."
  2. **첫 고객까지** — "만들고 끝이 아니라 유저를 데려옴."
- **금지:** "AI로 만들어드림"을 메인 메시지로 쓰지 말 것(커머디티화됨). 메인은 항상 *증거 + 첫 고객.*
- **타겟(작업용):** 돈·사업에 진심인 20~40대 중 *이미 아이디어 있고 지금 시작할 준비된* 사람(자영업자·1인사업가·부업러 포함). "관심만 있는" 드리머는 제외 → 카피로 자가선별.

---

## 3. 첫 빌드 스펙 (이번에 만들 것)

**서비스 랜딩 + 검증 퍼널:**

1. **랜딩 페이지** — 아래 섹션 순서:
   히어로(후크+CTA) → 문제(아이디어 있는데 막힘) → AI 함정 → 우리 방식(검증→제작→첫고객) → 창업자 스토리(득템잡이) → 오퍼 3단계 → 리스크 리버설 → FAQ → 최종 CTA
2. **리드·의도 캡처** — 폼: 이름 / 이메일 / 한 줄 아이디어 / 예산대. ("무료 상담/사전신청" 프레이밍 — 단순 이메일보다 강한 의도 신호)
3. **애널리틱스** — GA4 + 전환 이벤트(CTA 클릭, 폼 제출) + Google Ads 전환 태그 연결.
4. **리드 저장** — Supabase 테이블: `leads(id, name, email, idea, budget, source, created_at)`

## 5. 랜딩 카피 자산 (초안 — 다듬어 써도 됨)

**히어로 후크 (A/B 후보):**
- "아이디어는 있는데, 사이트·결제·마케팅에서 막히셨죠?"
- "안 될 것 같으면, 만들지 말라고 말해드립니다."
- "아이디어만 있으세요. 첫 고객까지, 저희가 데려옵니다."

**AI 함정:**
> "AI가 다 해주는 시대 같죠? 시장조사도, 사이트도 AI가 합니다. 근데 *진짜 수요가 있는지*는 AI가 안 알려줘요 — 어떤 아이디어든 '좋다'고 하거든요. 그 착각이 한 달을 태웁니다."

**창업자 스토리:**
> "저도 똑같이 했습니다. AI 믿고, 시장 분석 맡기고, 감으로 시작 — 한 달 넘게 끌렸고, 고객 반응은 없었고, 결국 접었어요(득템잡이). 그래서 이걸 만들었습니다. 당신은 그 한 달을 안 날리게."

**오퍼 3단계:**
1. **검증** — "될지 먼저 본다" (실제 광고+데이터로 수요 확인)
2. **런칭** — "되는 거면 만들고 첫 고객까지"
3. **성장(옵션)** — 월 리테이너로 계속 운영

**리스크 리버설:**
> "검증에서 수요가 안 나오면, 빌드 안 들어가고 환불합니다."

## 6. 가드레일 (빌드·카피 둘 다 적용)

- **소득·수익 보장 문구 절대 금지** (표시광고법 + 광고 계정 정지). "첫 결제"는 *목표·약속*으로 쓰되 매출을 *보장*하지 않음.
- **AI 생성 콘텐츠 표시 의무** 준수 (이미지/영상 사용 시).
- 결제 붙일 땐 **사업자등록 + 통신판매업 신고** 선행 (소규모 면제 기준 확인).
- 스코프: 웹/웹앱까지. 네이티브 앱 제외.
- 개인정보 수집 → **개인정보처리방침 + 이용약관** 페이지 필요.

## 7. Claude Code 첫 프롬프트 (복붙용)

```
위 브리프를 기반으로 Next.js(App Router) + Tailwind 랜딩 페이지를 만들어줘.

섹션 순서: 히어로(후크+CTA) / 문제 / AI 함정 / 우리 방식(검증→제작→첫고객) /
창업자 스토리 / 오퍼 3단계 / 리스크 리버설 / FAQ / 최종 CTA.

폼: 이름·이메일·한줄아이디어·예산대를 받아 Supabase의 leads 테이블에 저장.
GA4와 전환 이벤트(CTA 클릭, 폼 제출)를 붙여줘.

카피는 브리프의 초안을 쓰되 더 매끄럽게 다듬어도 돼.
모바일 우선, 신뢰감 있는 깔끔한 디자인. 과장된 톤 금지.
```

---

*핵심 다시: 빌드는 원가, 검증과 유통이 가격. 랜딩도 "AI로 만들어요"가 아니라 "증거 드립니다 + 안 될 건 말려드립니다"로.*
